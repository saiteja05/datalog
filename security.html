MongoDB Atlas Security

A Fortress for Your Data

MongoDB Atlas provides a fully managed, secure database service on AWS, Azure, and Google Cloud. Explore the comprehensive security features that protect your data at every layer, enabling you to develop rapidly while maintaining the highest standards of security.

Defend: Infrastructure & Data Protection

Network Isolation & Connectivity

  • VPC Isolation: Dedicated clusters are deployed in a fully isolated VPC/VNet.
  • IP Access List: Restrict cluster access to trusted IP addresses.
  • VPC Peering: Connect your application's virtual network directly to Atlas.
  • Private Endpoints: Ensure traffic never traverses the public internet using AWS PrivateLink, Azure Private Link, or Google Private Service Connect.
Learn More →

Encryption at Rest

  • Always-On Disk Encryption: Transparently encrypts all data on disk using AES-256.
  • Bring Your Own Key (BYOK): Encrypt data using your own keys from AWS KMS, Azure Key Vault, or GCP KMS for enhanced control.
Learn More →

Encryption in Transit

TLS 1.2+ is required for all connections to your Atlas cluster, protecting your data as it moves. Uses OpenSSL FIPS module for strong cryptographic standards.

Learn More →

Client-Side Field-Level Encryption

Encrypt specific fields on the client side before they are sent to the database, ensuring data is unreadable to anyone without the client keys.

Learn More →

Queryable Encryption

Perform queries on fully encrypted data while it remains unreadable to the server, allowing rich application functionality on your most sensitive data.

Learn More →

Secure by Default Configuration

Atlas is designed with strong security defaults, including always-on authentication, authorization, and encryption. Bastion hosts with MFA and strict approvals are required for any potential backend access by MongoDB personnel.

Learn More →

Authenticate & Authorize: Controlling Access

Atlas Control Plane Authentication

  • Multi-Factor Authentication (MFA): Require a second factor (app, device, SMS) for all users to log in to the Atlas UI.
  • Identity Federation (SSO): Integrate with your IdP (e.g., Okta, Azure AD) via SAML or OpenID Connect.
Learn More →

Database Authentication

  • SCRAM: Default username/password authentication (NIST 800-63 compliant).
  • X.509 Certificates: Authenticate using client certificates from your own PKI.
  • LDAP: Integrate with your corporate LDAP or Active Directory.
  • AWS IAM: Use AWS IAM roles for passwordless authentication.
Learn More →

Authorization (RBAC)

Implement the principle of least privilege with Role-Based Access Control. Use built-in or custom roles to grant specific permissions to users at the Organization and Project levels.

Learn More →

Secrets Management

Integrate with HashiCorp Vault to dynamically manage database credentials and reduce the risk of credential leakage.

Learn More →

Privileged User Access

MongoDB personnel do not have default access to customer clusters. Access by a small group of Privileged Users is gated, requires MFA, management approval, and is limited to the minimum time necessary to resolve critical issues. All access is logged and reviewed.

Learn More →

Detect & Monitor: Accountability and Logging

Database Auditing

Capture all administrative and DML/DDL activities with granular filters. Track actions by users, roles, and LDAP groups. Audit logs can be downloaded via UI/API for analysis and integration with SIEM tools like Splunk, Imperva, and Guardium.

Learn More →

Organization & Project Activity Feeds

Track all administrative actions taken within your Atlas Organization and Projects through the UI or API. The activity feed provides a chronological, filterable log of events like user access changes, cluster modifications, and network configuration updates.

Governance & Compliance Impact with Atlas

Security Governance & Risk Management

  • Regular Audits: Cloud provider audits are conducted regularly to validate compliance.
  • Vulnerability Management: Structured program using CVSS for risk-based remediation.
  • Threat Identification: As a Mitre CVE Numbering Authority, MongoDB helps identify and report threats.
  • Penetration Testing: Annual tests validate platform defenses.
Learn More →

Shared Responsibility Model

Atlas minimizes the customer's security burden with strong defaults like always-on authentication and encryption. The customer remains responsible for securely configuring their side, including network access rules, user permissions, and application-level security.

Learn More →

Resilience & Business Continuity

  • Self-Healing Replica Sets: Ensure high availability and automatic failover.
  • Backup & Restore: Easily distribute backup snapshots across multiple geographic regions.
  • BCDR Plan: A comprehensive plan aligned with ISO 22301 ensures tested RPOs and RTOs for service continuity.
Learn More →

Impact of Security & Compliance Add-ons

Total Cost of Ownership (TCO) is calculated factoring in advanced monitoring, security, and compliance capabilities that support key frameworks such as ISO, SOC 2, and PCI DSS. The estimate incorporates an industry-standard benchmark of 5% of total infrastructure cost to account for these add-on services.
