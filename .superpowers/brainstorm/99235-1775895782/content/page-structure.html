How should the page be structured?

This shapes the entire interaction model — choose what fits the multi-audience goal

A — Cinematic Scroll

Single linear narrative. Sections unfold like chapters as you scroll — each section builds on the last. Best for storytelling and guiding all audiences through the same arc.

↓ Atlas memory layer → ↓ Magenta compute layer → ↓ Architecture → ↓ Use cases → ↓ Roadmap → ↓ CTA

B — Hub + Explorer

Hero overview sets the vision, then four interactive "portals" — Architecture, Use Cases, Developer Experience, Roadmap. Visitors self-select their depth and path.

Executive skims hero → clicks Vision. Developer clicks Architecture. Partner clicks Roadmap.

C — Tabbed Journey

Three audience tabs at the top: "The Platform" / "The Architecture" / "The Roadmap." Each tab tells a complete story for that reader persona. Switchable any time.

Buyers stay in Platform. Engineers dive into Architecture. Partners track Roadmap.
